# Fitness Web Service API

This document outlines the REST API endpoints available in the Fitness Web Service, how to use them, and the expected request/response payloads.

## Base URL
If you are running the application locally using PHP's built-in web server (started with `php -S localhost:8000 index.php`), the base URL for the API is:
```
http://localhost:8000
```

---

## 1. Utenti (Users)
Manage user profiles before creating their fitness plans.

### Get All Users List
* **Endpoint:** `GET /utenti`
* **Description:** Retrieves a partial list of all users (`id`, `nome`, `email`).

### Create a User (Registration)
* **Endpoint:** `POST /utenti`
* **Content-Type:** `application/json`
* **Description:** Registers a new user in the system.

**Request Body:**
```json
{
  "nome": "Mario Rossi",
  "email": "mario.rossi@example.com",
  "password": "mysecretpassword",
  "eta": 28,
  "sesso": "M"
}
```
*(Note: `sesso` must be one of: 'M', 'F', 'Altro')*

**Success Response (201 Created):**
```json
{
  "id": 1,
  "message": "Utente creato"
}
```

### Login
* **Endpoint:** `POST /login`
* **Content-Type:** `application/json`
* **Description:** Authenticate a user with email and password.

**Request Body:**
```json
{
  "email": "mario.rossi@example.com",
  "password": "mysecretpassword"
}
```

**Success Response (200 OK):**
```json
{
  "message": "Login completato",
  "user": {
    "id": 1,
    "nome": "Mario Rossi",
    "email": "mario.rossi@example.com"
  }
}
```

**Error Response (401 Unauthorized):**
```json
{
  "error": "Credenziali non valide"
}
```

### Get User Details
* **Endpoint:** `GET /utenti/{id}`
* **Description:** Retrieves complete information about a specific user.

### Update a User
* **Endpoint:** `PUT /utenti/{id}`
* **Content-Type:** `application/json`
* **Description:** Completely updates an existing user profile.

### Delete a User
* **Endpoint:** `DELETE /utenti/{id}`
* **Description:** Deletes a specific user from the system.

## 2. Quiz (Fitness Questionnaire)
Store the user's fitness goals, level, and preferences. You need a valid `utente_id` to create a quiz.

### Submit a Quiz
* **Endpoint:** `POST /quiz`
* **Content-Type:** `application/json`
* **Description:** Saves the user's answers to the fitness questionnaire.

**Request Body:**
```json
{
  "utente_id": 1,
  "obiettivo": "massa",
  "livello": "intermedio",
  "giorni_settimana": 4,
  "durata_sessione": 60,
  "attrezzatura": "manubri, panca",
  "limitazioni": "leggero fastidio alla spalla sinistra"
}
```
*(Notes:)*
* `obiettivo` options: `'dimagrimento'`, `'massa'`, `'resistenza'`, `'mobilita'`, `'forza'`
* `livello` options: `'principiante'`, `'intermedio'`, `'avanzato'`

**Success Response (201 Created):**
```json
{
  "success": true,
  "message": "Quiz saved successfully.",
  "id": 1
}
```

### Get User's Quiz
* **Endpoint:** `GET /quiz/{id}`
* **Description:** Fetches the specific fitness quiz preferences submitted by the user.

---

## 3. Schede (Workout Plans)
Workout plans are generated based on the quiz answers. They are strictly linked (1:1) to a specific `quiz_id`.

### Create / Save a Workout Plan
* **Endpoint:** `POST /schede`
* **Content-Type:** `application/json`
* **Description:** Saves an AI-generated workout plan into the database.

**Request Body:**
```json
{
  "utente_id": 1,
  "quiz_id": 1,
  "titolo": "Scheda Massa Intermedio - 4 Giorni",
  "contenuto": "{\"giorno_1\": [\"panca piana\", \"croci\"], \"giorno_2\": [\"squat\", \"affondi\"]}",
  "modello_ai": "google/gemini-pro"
}
```
*(Note: `contenuto` is meant to store the raw JSON string or text structure generated by the AI).*

**Success Response (201 Created):**
```json
{
  "success": true,
  "message": "Workout plan saved successfully.",
  "id": 1
}
```

### Get Workout Plan
* **Endpoint:** `GET /schede/{id}`
* **Description:** Retrieves the workout plan associated with a specific id.

## Testing the API with cURL

You can quickly test these endpoints using your terminal once your server is running. Here is an example of creating a user:

```bash
curl -X POST http://localhost:8000/utenti \
     -H "Content-Type: application/json" \
     -d '{"nome":"Luca Bianchi","email":"luca@example.com","eta":25,"sesso":"M"}'
```
